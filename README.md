## 🚀 Shipped Work

### AI Innovation Platform — Africa's Largest Bank
Started with an empty subscription. Ended with a production-ready Azure Landing Zone serving thousands of bank employees across five subsidiaries. Every resource provisioned via Terraform — VNets, private endpoints, App Services, Cosmos DB, Redis Cache, Key Vault — wired together with Azure DevOps pipelines and an AI layer built on Azure AI Foundry for LLM-assisted scoring. Nothing was ClickOps. Every environment (dev, test, prod) reproducible and version-controlled from day one.

`Azure Landing Zone` `AKS` `Azure AI Foundry` `Cosmos DB` `Terraform` `Azure DevOps` `Private Endpoints` `Entra ID / SSO`

---

### SAP Order-to-Delivery Platform — Global FMCG Enterprise
Not a side project — the order management backbone for one of Africa's largest conglomerates, processing thousands of transactions daily across a nationwide distribution network. Built the Azure infrastructure connecting SAP to the cloud: event-driven messaging via Service Bus and Event Hubs, secure private networking between on-prem SAP systems and Azure, and AKS clusters running the microservices layer. Had to be highly available, auditable, and resilient enough for enterprise-grade SLAs.

`SAP Integration` `Service Bus` `Event Hubs` `Apache Kafka` `AKS` `Hub-Spoke VNet` `ExpressRoute` `RBAC Governance`

---

### Next-Gen AI Digital Banking Platform — Nigeria's First Digital Bank
Nigeria's first fully digital bank needed to evolve beyond its original architecture — the mobile SDK was end-of-life, performance was hitting limits at scale, and the next version had to support AI-driven credit decisions, fraud detection, and personalized banking across web, mobile, USSD, and chatbot channels. Contributed the cloud infrastructure design: AKS-hosted microservices, APIM-fronted consumption layer, Prometheus/Grafana observability, and a DR architecture replicating data in real-time between primary DC and cloud via ExpressRoute.

`AKS Microservices` `Azure APIM` `Prometheus / Grafana` `DR Architecture` `ExpressRoute` `AI Decision Layer`

---

### Zero-Downtime Azure Region Migration — Financial Services
Four platform-critical services. No native migration path for any of them. Planned and executed the full move West Europe → North Europe using a Rebuild + Reconfigure + Reconnect strategy — new Notification Hub namespaces, exported and reimported registrations, migrated Log Analytics workspaces with all custom KQL queries and dashboards intact, reconnected CI/CD pipelines without dropping a single build. Total cutover in a maintenance window. Every alert rule fired correctly and all dependent apps reconnected first try.

`Azure DevOps` `Notification Hubs` `Log Analytics / KQL` `Services Hub` `Zero Downtime`

---

### Azure SME — 200+ Enterprise Customers (Microsoft)
For nearly three years, owned complex Azure cases from open to resolution — Application Gateway misconfigurations, AKS CNI and Cilium networking breakdowns, VPN Gateway tunnel failures, ExpressRoute BGP issues, Front Door routing anomalies. Used Log Analytics, Azure Support Center, and packet captures to isolate root causes that most engineers would escalate. That period built an instinct for Azure's internals that certifications alone don't give you.

`App Gateway / AGIC` `AKS CNI` `VPN Gateway` `ExpressRoute` `Front Door` `DNS / Bastion` `RCA & Log Analytics`
